AI-Powered Conversation Chatbot

An automated chatbot system built to handle customer inquiries and schedule appointments through natural conversations utilizing LLM's.

Project Goals

This project aims to streamline customer interactions using AI and telephony services. It allows users to engage with the chatbot via phone calls, respond to queries, and even schedule appointments by leveraging ChatGPT and a web-based audio framework for efficient conversation handling.

Phone Connection

The AI chatbot connects to real users through telephone, analyzing voice commands and just as easily responding.

API Integration

The chatbot uses multiple APIs, such as Azure for voice recognition, ElevenLabs for voice generation, and Google Calendar API to retrieve available times and confirm appointments with customers.

Key Features

  • Natural Language Processing for real-time conversations
  • Twilio API integration for call handling
  • Google Calendar integration for scheduling appointments
  • Custom audio prompts and Text-to-Speech (TTS) responses
  • Flexible architecture allowing local or cloud-based deployment

How It Works

Once a call is initiated, the system starts with an introductory message, handles customer queries, and guides them through a conversation flow based on predefined intents (e.g., asking questions, scheduling appointments). By utilizing neural networks, the system can accurately interpret customer inputs and provide dynamic responses.

A RAG process using TF-IDF performs efficient querying on scraped web-pages -- in this case, 150+ scraped company web pages.

Conversation Flow

Twilio integrations enable a structured flow for the customer's call, from greeting to handling specific inquiries and finalizing appointments or call terminations.

Text-to-Speech & Recognition

Using ElevenLabs TTS and Azure audio recognition, the system can convert text to lifelike speech to answer questions and accurately process customer responses.

Additional Capabilities

The chatbot supports a variety of capabilities, including:

  • Handling invalid responses and repeating prompts when necessary
  • Logging call information for further analysis and improvements
  • Flexible audio file management for customized responses

Deployment

Maintained a Docker image to deploy as an Azure App Service utilizing CI/CD for seamless updates and scalability.
